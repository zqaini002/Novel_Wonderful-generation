# 小说精读助手数据库

本目录包含小说精读助手应用的数据库相关文件，用于创建和初始化数据库结构及示例数据。

## 文件说明

1. **schema.sql** - 数据库表结构定义
   - 创建数据库及所有表
   - 定义表关系、约束和索引
   - 使用UTF-8编码支持中文字符

2. **sample_data.sql** - 示例数据插入脚本
   - 包含两部示例小说：修真类《修真世界》和科幻类《未来科技城》
   - 每部小说包含5章内容，带有摘要、关键词和分析数据
   - 提供完整的标签、风格特征、题材特征等分析示例

3. **er_diagram.md** - 数据库ER图及关系说明
   - 使用ASCII图表展示实体关系
   - 详细说明各表之间的关系
   - 解释数据流处理过程

## 使用方法

1. 创建数据库结构：
   ```bash
   mysql -u username -p < schema.sql
   ```

2. 导入示例数据：
   ```bash
   mysql -u username -p < sample_data.sql
   ```

## 数据库结构概述

小说精读助手数据库设计遵循以下核心原则：

1. **小说内容层次化存储**
   - 小说→章节→摘要/关键词/人物
   - 支持章节级别的内容分析

2. **分析结果多维度展现**
   - 风格特征（如场景描写、情感刻画等）
   - 题材特征（如玄幻、科幻、悬疑等）
   - 潜在问题（如情节冗长、人物单薄等）
   - 章节建议（如增加角色发展、减少过渡章节等）
   - 内容推荐（如继续阅读、相似内容等）

3. **小说分类标签体系**
   - 推荐标签：积极特点（如修真升级、科技感强等）
   - 避雷标签：潜在问题（如套路化情节、技术术语多等）
   - 阅读门槛标签：需要的前置知识（如修真知识等）

## 字符编码说明

数据库及所有表使用`utf8mb4`字符集和`utf8mb4_unicode_ci`排序规则，以支持完整的中文字符及表情符号。
